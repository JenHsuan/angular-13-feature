Traditional web animations
There are two traditional ways to generate animations.
  • Generate animations from the specific CSS pusedo classes when there are new events (e.g, hover).
  • Generate animations through manipulating the CSSDOM from JavaScript.
The purpose of the native web animation API is able to generate animations by manipulating DOM with JavaScript.
Compatibility
Most modern browsers support the native web animation API.
How does it work?
  • Component
    1. Define objects in keyframes format . There are two ways.
      • Create an array of keyframe objects to iterate over
      • Create an object containg key-value pairs consisting of the property to animate and an array of values to iterate over
    2. Specify the KeyframeEffect() options parameter and use Element.animate() to generate the animation.
      • duration: the number of milliseconds each iteration of the animation takes to complete. (defaults to 0)
      • iteraion: the number of times the animation should repeat. (defaults to 1)
      • easing: the rate of the animation's change over time. Accepts an easing-function .
      • delay: the number of milliseconds to delay the start of the animation. (defaults to 0)
  • Add two images on the template. (original-logo and the reverted-logo)
    Bind the switchImage function to the image on the template
  • Add the grayscale filter on the original logo
