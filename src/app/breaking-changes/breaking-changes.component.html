There are a few breaking changes in v13.
  1. v13 requires node.js version v12.20.0 (Version Compatibility)
  2. v13 requires TypeScript version 4.4 or later (Version Compatibility)
  3. v13 uses RxJS 7.4
  4. Stop supporting IE 11 (Pull request)
    There are a few benefits.
    1. Angular can remove the IE-specific code paths (JavaScript and CSS), polyfills, build passes, and infrastructure.
    2. The differential loading feature has been removed.
  5. Ivy everywhere: View engine is removed and Ivy is fully used (also check the note of Angular 12)
    There are a few benefits.
    1. We no longer needs to be compile libraries (built with APF 13) with ngcc.
    2. Developers can create dynamic components easier by using the new component APIs
    3. Developers are allowed to create animations through the native API.
      (clarification: CSS variables is already enabled in Angular 12)

With the differential loading, Angular can separate JavaScript code to different bundles and include them.
  • The type="nomodule" attribute addresses this bundle does not belong to EcmaScript module (ESM).
    For the legacy browsers (e.g., IE 8), they can only recognize type="nomodule" attribute.
  • The type="module" attribute addresses this bundle belongs to EcmaScript module (ESM).
    For the modern browsers, they can recognize type="module" attribute.
  • Now, Angular only use type="module" attribute because it has stopped supporting IE 11.

The following diagram shows the workflow of asking server for the web page from the browser. The rendering will be blocked by the critical paths.
  • Process of downloading JavaScript files.
  • Process of downloading CSS files.
  • Process of executing JavaScript files. (e.g., Document.createElement API)
  • Process of creating CSSDOM
(taken from this reference)
There are a few methods to avoid render blocking during the JavaScript downloading and executing.
  1. After adding type="module" attribute, JavaScript files will be downloaded asynchronously and be executed after the rendering is completed.
    In Angular, it's also used for being recognized if the bundle could support the modern browsers.
  2. After adding the async attribute, JavaScript files will be downloaded asynchronously and be executed before rendering completed.
