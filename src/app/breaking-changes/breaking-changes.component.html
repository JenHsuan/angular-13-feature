There are a few breaking changes in v13.
  1. v13 requires node.js version v12.20.0 (Version Compatibility)
  2. v13 requires TypeScript version 4.4 or later (Version Compatibility)
  3. v13 uses RxJS 7.4
  4. Stop supporting IE11 (Pull request)
    There are a few benefits.
    1. Angular can remove IE-specific JS and CSS code paths, polyfills, build passes, and dev infrastructure.
    2. The differential loading has been removed.
  5. Ivy everywhere: remove View engine and fully use Ivy (check the note of Angular 12 for details)
    There are a few benefits.
    1. Libraries built with APF 13 no longer needs to be compiled with ngcc.
    2. The component API for creating dynamic components has been improved.
    3. The animation from native API has been enabled. (clarification: CSS variables is already enabled in Angular 12)

The ability of differential loading is to include different bundles according to different browsers. It depend on the browser's compatibility.
  • Previously, we have to include polyfills to support legacy browsers. (e.g., Angular 8)
    • For the legacy browsers (e.g., IE 8), we can use type="nomodule" because it can't support EcmaScript modules (ESM).
    • For the modern browsers, we can use type="module" to address it's ESM.
  • Now, we can only use type="module" in the index.html because the Angullar 13 stops supporting IE 11.

The folloing diagram shows the workflow between the browser ask the web page and the browser display the recived we page. The rendering will be blocked by the critical paths.
  • Download JavaScript files.
  • Download CSS files.
  • Execute JavaScript files. (e.g., Document.createElement API)
(taken from this reference)
There are a few methods to avoid render blocking during the JavaScript downloading and executing.
  • With adding type="module" attribute, JavaScript files will be downloaded asynchronously and be executed after the rendering is completed.
    In Angular, it's also used for being recognized if the bundle could support the modren browsers.
  • With adding the async attribute, JavaScript files will be downloaded asynchronously and be executed before rendering completed.
