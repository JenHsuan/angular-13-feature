There are a few breaking changes in v13.
  1. Angular v13 requires node.js version v12.20.0 (Version Compatibility)
  2. Angular v13 requires TypeScript version 4.4 or later (Version Compatibility)
    • Improvement of type guard
    • Defaulting to the unknown Type in Catch Variables if we enable useUnknownInCatchVariables
    • Symbol and template string pattern index signatures
    • static Blocks in Classes
  3. Angular v13 uses RxJS 7.4
  4. Stop supporting IE11 (Pull request)
    • The IE-specific polyfills were dropped
    • Be able to remove special JS and CSS code paths, polyfills, build passes, and dev infrastructure that exists
    • Be able to use CSS Variables, Intersection Observers, CSS Grid after removing IE11 supports
  5. Remove differential loading: support different browsers with different bundles
    • When we want to import ES6 modules, we can use type="module" to let the browser knows
    • For the legacy browsers (e.g., IE 11), it can't support ES6 modules so we use nomodule

The rendering will be blocked when the scrippt is downloaded or executed. (synchronous)
There are a few methods to avoid render blocking
  • The script will be asynchronously downloaded when specifying the module attribute. The script will be executed after rendering completed.
  • The script will be asynchronously downloaded when adding the async attribute. The script will be executed before rendering completed.
