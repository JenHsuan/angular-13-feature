There are a few breaking changes in v13.
  1. v13 requires node.js version v12.20.0 (Version Compatibility)
  2. v13 requires TypeScript version 4.4 or later (Version Compatibility)
  3. v13 uses RxJS 7.4
  4. Stop supporting IE11 (Pull request)
    There are a few benefits.
    1. The IE-specific polyfills were dropped
    2. Be able to remove special JS and CSS code paths, polyfills, build passes, and dev infrastructure that exists
  5. Ivy everywhere: remove View engine and fully use Ivy (check the note of Angular 12 for details)
    There are a few benefits.
    1. Librarie built with Angular Package Format (APF) in v13 no longer need to be compiled with ngcc.
    2. The component API for creating dynamic components has been improved.
    3. The CSS variables and native web animation API has been enabled.
    4. The differential loading feature has been removed because the Angular 13 stops supporting IE 11.

The rendering will be blocked in the following situations. The rendering process and them are synchronous to each other.
  • The JavaScript file is downloaded.
  • The CSS file is downloaded.
  • The JavaScript file is executed.
(taken from this reference)
There are a few methods to avoid render blocking
  • The script will be asynchronously downloaded when specifying the module attribute. The script will be executed after rendering completed.
  • The script will be asynchronously downloaded when adding the async attribute. The script will be executed before rendering completed.

The ability of differential loading is to upport different browsers with different bundles.
  • Previously, we have to use polyfills to support legacy browsers. (e.g., Angular 8)
    • For the legacy browsers (e.g., IE 8), it can't support ES6 modules so we can only use type="nomodule".
    • For the modern browsers, we can use type="module" to represent these modules are ES6 modules.
  • Now, we only use type="module" in the index.html because the Angullar 13 stops supporting IE 11.
